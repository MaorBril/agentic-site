Features

Two questions Claude Code never answered.

How much did that session cost? And can I run the cheap parts on a cheap model? agentic answers both — without touching Claude Code itself.

◇

Any model, any provider

Anthropic passthrough is byte-faithful. OpenAI, xAI, Ollama, vLLM, OpenRouter and more work through full request/stream translation. Model names are aliases you define.

$

Every token metered

A local SQLite usage log prices every request. Run agentic cost to break spend down by model, profile, or session — with a live figure in your statusline.

⛨

Budgets that bite

Daily, weekly, and monthly caps — global or per profile. Hit a cap and the router refuses the next request with a clear message, right in the TUI. In-flight responses are never cut.

⟲

Dynamic routing

Let a cheap classifier triage every task: planning goes to a big model, mechanical edits to a cheap one. The decision sticks for the whole turn — no flip-flopping mid-task.

◎

No daemon, no fork

The first session binds the router port and serves everyone; when it exits, another takes over in seconds. Claude Code stays unmodified and keeps auto-updating.

⌘

Profiles & overrides

Bundle a main model, a small/fast model for background tasks, and budgets into a profile. Switch with -p cheap or override once with --model grok.

How it works

A gateway, plus the CLI around it.

Claude Code officially supports pointing at a gateway via ANTHROPIC_BASE_URL. agentic is that gateway.

1

Install & set up

One curl command drops the binary in ~/.local/bin. agentic setup writes a config, registers the statusline, and prompts to install any missing deps.

2

Launch a session

Run agentic. It starts the local router, points Claude Code at it via ANTHROPIC_BASE_URL, and hands you the exact same TUI.

3

Route, meter, cap

The router resolves your model alias, translates the dialect if needed, logs priced usage, and gates on your budget — then streams the response straight back. 

# Claude Code, tracked, on your default profile
❯ agentic

# same session, cheaper models
❯ agentic -p cheap

# one-off model override
❯ agentic --model grok

FAQ

The fine print.

How is this billed?

Traffic through the router is billed to your API keys, not your Claude Pro/Max subscription. OAuth credentials are never proxied. For subscription billing, use a passthrough: true profile — normal claude, no tracking.

Do non-Anthropic models work as well?

They work through translation, but Claude Code's prompts and tool patterns are tuned for Claude, so expect them to be clunkier in the main loop. They shine as cheap workhorses for background tasks and subagents.

Is Claude Code modified?

No. agentic launches the unmodified, auto-updating claude binary and points it at the local router via ANTHROPIC_BASE_URL — an officially supported gateway hook.

Is it secure?

The router binds 127.0.0.1 only and requires a per-install token (mode 0600), so other local processes can't spend on your keys. Keys live in ~/.agentic/env, never in the config file.

What does it cost?

agentic is free and MIT-licensed. You only pay your model providers for the tokens you use — which is exactly what agentic helps you measure and cap.
